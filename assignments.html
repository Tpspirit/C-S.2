Platform


There were many choices how to handle the team web portfolio. After a few minutes of discussion, our decision was to use HTML & CSS to design our web portfolio. Actually, we could take advantage of some frameworks or existing library, however, in order to practice and everyone is able to contribute to the assignment, we decided to not use any framework but only HTML & CSS files. There are almost no obstacles as we have been already familiar to website creation. Requirements and contents are created by HTML, while CSS is used to display and animate how everything appears on our website.


To make it online, we host the web portfolio on GitHub Pages. Collaboration is also available within this platform so all members could add or make changes at any time. GitHub offers no challenges when we update anything, thus, the assignment is always available to finish for everyone.


Tools


For demonstrating the layout of our application from both the user and developer perspectives we have utilized platforms such as TheyDo, Miro and Visual Paradigm. TheyDo is a free “Journey Management” platform which provides a variety of tools for simple yet very detailed journey mapping. Miro is another simple platform which is primarily famous for it’s whiteboarding and that’s where we created our service blueprint. Visual Paradigm is a niche application/platform that has a vast amount of functions and is used for project management, we used it to quickly create our empathy map.


For designing application interfaces, our team settled on Figma. The variety of features and tools available can significantly speed up design process. With the use of font styles, color schemes, and other aspects, users can construct their own design systems. Figma enables team collaboration while simultaneously displaying what other team members are working on.
